# Git Workflow

This page declares the lifecycle of development and naming conventions for commits and branches

## Order of actions to edit code

1. Create In Issue
2. Create a new branch
3. Resolve the issue
4. Commit commit and push
5. Create Pull Request
6. Merge to main

---

## Branch naming convention

- feature/<name>
- fix/<name>
- refactor/<name>
- chore/<name>
- docs/<name>

**Examples**

- feature/cookie-modal
- fix/CTA_button
- refactor/carousel
- chore/add-babel
- docs/project-structure


## Commit naming convention
<type>(<scope>): <description>

**Examples**

- feat(cookie-model): added glowing effect
- fix(CTA_button): fixed spacing
- refactor(carousel): removed code duplicates
- chore(add-babel): installed babed
- docs(project-structure): added babel setup instructions
